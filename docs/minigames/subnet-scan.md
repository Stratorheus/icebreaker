# Subnet Scan (`subnet-scan`)

## Player Guide

A CIDR subnet range (e.g., `192.168.5.0/24`) is displayed at the top of the screen. Below it, a list of IP addresses is shown. You must select all addresses that belong to the displayed subnet. Selecting a wrong address causes immediate failure. A help box at the bottom explains how the current subnet mask works. Select all correct addresses to win.

## Mechanic

**Puzzle generation:**

1. A random CIDR prefix is chosen from the difficulty-appropriate pool.
2. A random private IP range base is selected (10.x.x.x, 172.16-31.x.x, or 192.168.x.x).
3. The network address is computed by zeroing host bits: `networkInt = rawIp & mask`.
4. `correctCount` addresses within the subnet are generated by adding random host parts (avoiding .0 network and .255 broadcast), verified with `isInSubnet()`.
5. `totalAddresses - correctCount` decoy addresses are generated using three strategies:
   - Offset a network-portion octet by +/- 1-3.
   - Use a host address just outside the subnet boundary.
   - Same first octets but different network-significant octet.
6. All addresses are shuffled together.

**Subnet math:**
- `isInSubnet(ip, network, prefix)`: compares `ip & mask === network & mask` where `mask = 0xFFFFFFFF << (32 - prefix)`.

**Resolution:**
- Selecting a wrong (out-of-subnet) address: immediate failure after 400 ms visual delay.
- Selecting a correct address: increments counter. When all correct found, success after 400 ms.
- Deselecting is allowed with no penalty.
- Timer expiration: failure.

## Difficulty Scaling

| Parameter | d < 0.3 | d = 0.3 - 0.6 | d > 0.6 | Formula |
|-----------|---------|---------------|---------|---------|
| Total addresses | 4 | 6 | 6 | Stepped |
| Correct count | 1 | 2-3 | 3-4 | `round(2 + d)` / `round(3 + (d-0.6)*2.5)` capped at 4 |
| Prefix options | /8, /16, /24 | /16, /24 | /20, /22, /24 | Stepped pools |

At low difficulty, easy-to-reason prefixes (/8, /16, /24) are used where full octets must match. At high difficulty, partial-octet prefixes (/20, /22) require bit-level reasoning.

## Controls

### Desktop

- **Arrow Up / Arrow Down** -- move keyboard cursor through the address list.
- **Space** -- toggle select/deselect on the focused address.
- **Mouse click** -- toggle any address directly.

### Mobile

- **Tap** -- toggle select/deselect on any address.

## Base Time Limit

**20 seconds.**

Scaling formula: `round(20 * (1 - difficulty * 0.4) * floorScale * 1.03^timerExtTier)`

Where `floorScale = max(0.4, 1 - (floor - 15) * 0.02)` for floors > 15, otherwise 1.

Additional timing modifiers that affect the effective timer:
- **Time Siphon** (run shop): +0.2 s per consecutive win (floor-scoped, resets on fail).
- **Cascade Clock** (meta upgrade): +2% of base timer per consecutive win (cap per tier, resets on fail, persists across floors).
- **Deadline Override** (run shop): injects +1 s when timer drops below 5% (single use).
- **Time-bonus** power-ups: flat seconds added by `useMinigame` hook on mount.

## Code Reference

**Component path:** `src/components/minigames/SubnetScan.tsx`

**Key functions:**
- `ipToInt(ip)` / `intToIp(n)` -- convert between dotted-decimal strings and 32-bit integers.
- `isInSubnet(ip, network, prefix)` -- bitwise subnet membership check.
- `getParams(difficulty)` -- returns `totalAddresses`, `correctCount`, `prefixOptions` per difficulty band.
- `generatePuzzle(params)` -- full puzzle builder: picks CIDR, generates correct + decoy IPs.
- `generateNearbyOutOfSubnet(networkInt, prefix, baseOctets)` -- creates plausible-looking out-of-subnet IPs.
- `getMaskHelp(prefix)` / `getMaskDetail(prefix)` -- human-readable subnet mask explanations.
- `handleToggle(index)` -- select/deselect logic with win/fail resolution.

**State variables:**
- `selectedSet: Set<string>` -- currently selected IP addresses.
- `correctCount: number` -- number of correctly selected addresses.
- `cursorIndex: number` -- keyboard cursor position in the address list.
- `hasCidrHelper: boolean` -- whether the CIDR Helper power-up is active.
