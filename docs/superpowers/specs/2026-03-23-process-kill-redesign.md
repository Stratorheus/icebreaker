# Process Kill Redesign — Design Spec

## Goal

Reskin the Wire Cutting minigame to "Process Kill" — cyberpunk-themed matrix rain streams replacing physical wires. Visual-only change + naming update. No mechanic changes.

## Scope

- Visual: colored wire bars → animated matrix rain hex streams
- Naming: "Wire Cutting" → "Process Kill", "cut" → "terminate", "wire" → "process/stream"
- Upgrade: "Wire Guide" → "Stream Monitor"
- All references in briefing, codex, meta shop, achievements, tests

## 1. Visual Design

### Active Stream
- Width: 32px, height: 140px (increased from current 120px for better rain effect)
- No side borders. Top border: 2px solid at 45% stream color opacity. Bottom border: 2px at 25% opacity.
- Content: 3 overlapping columns of falling hex characters (matrix rain animation)
  - Column positions: left offsets 0, 6, 12px (overlap to create dense stream)
  - ~9 characters per column, falling top→bottom with CSS `@keyframes`
  - Duration: 1.6–3.0s per character (randomized)
  - Opacity: 0→0.8 (top) → 0.5 (middle) → 0 (bottom)
  - Characters randomly mutate every 150ms (~12% chance per char)
- Color: stream color applied to characters and borders
- Below stream: number (1-based) + color name label

### Killed Stream
- Animation freezes in place (`animation-play-state: paused`)
- Characters turn grey (`rgba(255,255,255,0.12)`)
- Top/bottom borders fade to `rgba(255,255,255,0.06)` / `0.04`
- Number and name label grey out
- No ✕ overlay, no removal — stream visually "dies" in place

### Next Target (Stream Monitor upgrade active)
- Non-target streams dim to 35% opacity (same as current `isDimmed`)
- Next target stream gets `box-shadow: 0 0 18px {color}` glow
- Keyboard hint for next target highlighted in stream color

### Hover State
- Top/bottom borders brighten to `rgba(255,255,255,0.4)` / `0.3`

## 3. Component Changes

### WireCutting.tsx → ProcessKill.tsx (optional rename)
Keep the file as `WireCutting.tsx` for git history clarity (or rename — both acceptable). The component export can stay `WireCutting` since it's referenced by the registry.

**Visual rendering changes:**
- Replace the colored bar div with a matrix rain container
- Each stream: outer div (32px wide) with top/bottom border only
- Inside: 3 absolutely-positioned rain columns with overlapping hex characters
- CSS animation `@keyframes fall` for top→bottom movement
- `setInterval` every 150ms to randomly mutate visible characters (12% chance each)
- Killed streams: add `killed` class → `animation-play-state: paused` + grey color
- Remove the diagonal "cut line" SVG overlay (replaced by freeze effect)

**Text changes in component:**
- "CUT {n} OF {m} WIRES ({total} total)" → "TERMINATED {n} OF {m} PROCESSES ({total} total)"
- Any other hardcoded "wire"/"cut" strings

### wire-cutting.ts (config)
- `displayName: "Process Kill"`
- Update all briefing strings (rules, controls, tips, hint)
- Update metaUpgrades name/description

## 4. What NOT to Change

- Game mechanic (rule generation, validation, scoring)
- Difficulty scaling (wire count formula, rule complexity)
- Color palette (same 8 colors)
- Keyboard/touch interaction (number keys, click/tap)
- Internal IDs (MinigameType, upgrade IDs, license ID, effect types)
- Store state, power-up effects system
- Timer, base time limit

## 5. Performance Notes

- Matrix rain uses CSS animations (GPU-accelerated), not JS requestAnimationFrame
- Character mutation via `setInterval` at 150ms is lightweight (~27 DOM text updates per tick for 3 cols × 9 chars = 27 chars per stream × 12% = ~3 actual updates per stream per tick)
- On 7 streams (max difficulty): ~21 text mutations per 150ms — negligible
